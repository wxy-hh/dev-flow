# /review-diff — 审查当前改动

用于审查当前工作区 diff，优先发现真实 bug、回归风险、安全问题和缺失验证。

## 使用

```text
/review-diff
```

## 流程

1. 查看 `git status --short`、`git diff --stat` 和当前 diff。
2. 阅读改动文件及必要的调用上下文。
3. 按严重程度审查：
   - CRITICAL：安全漏洞、数据丢失、鉴权绕过、构建必失败
   - HIGH：明确运行时错误、关键流程回归、接口契约错误
   - MEDIUM：边界情况、缺少错误处理、验证不足
   - LOW：可维护性问题，且必须与本次改动直接相关
4. 只报告有较高把握的问题，不输出风格偏好。

## 输出格式

```text
发现：
- [HIGH] 文件:行 问题说明、影响、建议修复

开放问题：
- ...

验证建议：
- ...
```

如果没有发现问题，明确说明“未发现阻塞问题”，并列出剩余验证风险。
