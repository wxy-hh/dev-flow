# Commands

| 命令 | 用途 |
|------|------|
| `/dev-task` | 新需求入口：先判 XS/S/M/L，再识别独立风险标签 |
| `/onboard-dev-flow` | 根据当前项目生成 0.4 适配配置 |
| `/review-diff` | 审查当前工作区真实改动 |
| `/finish` | 运行完成前审查、验证和 work checker |
| `/fix-build` | 最小修复类型或构建错误 |
| `/commit` | 在验证后整理提交建议；按用户授权提交 |

使用原则：

- 普通 XS/S/M 默认不生成流程文件。
- L 或任意风险任务只维护一个 `work.md`，独立手测确有必要时才增加附件。
- 高风险实现前必须经过风险卡之后的用户确认；最初的“直接改”不算确认。
- L 按批次执行并在每批后更新同一恢复点，不固定串联覆盖、审查或回撤报告。
- `/finish` 没有新鲜证据时不能给出“完成”结论。
- Git 提交、推送、合并、回滚和删除操作按用户明确授权执行。
