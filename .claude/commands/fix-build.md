# /fix-build — 修复类型和构建错误

用于让当前项目回到可类型检查或可构建状态，保持最小改动。

## 使用

```text
/fix-build
```

## 流程

1. 读取 `.claude/rules/project-workflow.md`，确认当前项目的类型检查和构建命令。
2. 运行适配层定义的类型检查命令收集错误。
3. 如涉及构建配置或产物，再运行适配层定义的构建验证。
4. 按错误类别定位：
   - 类型不匹配
   - 缺失导入或路径错误
   - 框架模板、组件或页面类型问题
   - 配置或依赖问题
5. 做最小修复，不顺手重构。
6. 重新运行失败过的验证命令，直到输出支持结论或明确剩余阻塞。

## 约束

- 不做无关重构。
- 不改业务行为，除非这是修复错误的必要条件。
- 不用 `as any` 或 `@ts-ignore` 掩盖问题。
