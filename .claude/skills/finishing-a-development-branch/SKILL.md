# 完成开发分支

## 收尾前

1. 查看当前分支、工作区状态和完整 diff。
2. 确认代码审查阻塞项已处理或被明确接受。
3. 确认验证发生在最后一次相关改动之后。
4. 有 `work.md` 时运行 `dev-flow-feature-check <feature-id> --finish`；失败则停止。

不要生成新的总结文件，不归档或删除现有工作资产。`work.md` 本身就是最终边界、进度、风险和验证记录。

## 用户选择

完成检查后询问用户希望：

1. 保留当前工作区/分支，不执行 Git 动作。
2. 提交当前改动。
3. 推送并创建 PR。
4. 合并到指定分支。
5. 丢弃改动或删除分支。

只有用户明确授权对应动作后才执行。推送、合并、rebase、丢弃改动和删除分支需要在执行前确认目标；禁止用破坏性命令替用户猜测。

最终汇报只包含实际改动、验证结果、残留风险和已执行的 Git 动作。
